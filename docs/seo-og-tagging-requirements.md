# SEO and OG Tagging Requirements

Use this checklist whenever a new public page is created or an existing page is prepared for indexing.

## Required Page Inputs

Every indexable page needs these fields before launch:

- Route path, for example `/product` or `/playbooks/cac-playbook`
- SEO title
- Meta description
- Canonical URL
- Open Graph title
- Open Graph description
- Open Graph image path
- Twitter card title
- Twitter card description
- Twitter image path
- Sitemap priority
- Sitemap change frequency
- Indexing decision: `index` or `noindex`

## Title and Description Guidelines

- SEO title should be specific to the page and end with the brand when useful.
- Home page title should be `I Want That! | Customer Generated Offer Management`.
- Keep meta descriptions focused on the page promise, not internal feature labels.
- Avoid duplicate titles and duplicate descriptions across public pages.
- If Open Graph copy can be stronger for social sharing, write it separately from SEO copy.

## Open Graph and Twitter Requirements

Every indexable marketing, tool, playbook, and content page should include:

- `openGraph.title`
- `openGraph.description`
- `openGraph.url`
- `openGraph.type`
- `openGraph.images`
- `twitter.card`
- `twitter.title`
- `twitter.description`
- `twitter.images`

Use `summary_large_image` for Twitter unless there is a specific reason not to.

## OG Image Requirements

Create or select an image before adding the page to the sitemap.

- Preferred size: `1200 x 630`
- Format: PNG or JPG
- Store site-owned static images in `public/`
- Use absolute URLs in metadata, for example `https://useiwantthat.com/vector-icon-wordmark-logo.png`
- Image should still read clearly when cropped in social previews
- Include the page/product/playbook subject, not a generic abstract background
- Do not reference image files that are not present in `public/`

## Canonical Rules

- Every indexable route should have a canonical URL.
- Canonical URLs should use the production domain.
- Duplicate route aliases should either redirect to the canonical route or use a canonical pointing at the preferred route.
- Legal and policy duplicate routes should not both be treated as primary indexable pages.

## Sitemap Rules

Add a route to `app/sitemap.ts` when the page is intended to be indexed.

Recommended sitemap fields:

- `url`
- `lastModified`
- `changeFrequency`
- `priority`

Suggested priorities:

- Home: `1.0`
- Product and core conversion pages: `0.9`
- Tools and playbooks: `0.8`
- Blog index and major hubs: `0.7`
- Standard content pages: `0.6`
- Legal pages: `0.3`

Suggested change frequencies:

- Home/product/tools/playbooks: `monthly`
- Blog index: `weekly`
- Blog posts: `monthly`
- Legal pages: `yearly`

## Robots and Noindex Rules

Use `noindex` for:

- Preview pages
- Duplicate route aliases that are not redirected
- Internal utility pages
- Pages without complete public content

Use `robots.ts` for sitewide crawler rules and sitemap location once that file exists.

## New Page Launch Checklist

- [ ] Page has final visible content.
- [ ] Page has a unique SEO title.
- [ ] Page has a unique meta description.
- [ ] Page has Open Graph title and description.
- [ ] Page has Twitter title and description.
- [ ] OG/Twitter image exists in `public/` or is generated by the app.
- [ ] Canonical URL is set.
- [ ] Indexing decision is explicit.
- [ ] Indexable page is included in `app/sitemap.ts`.
- [ ] Duplicate or legacy routes are redirected, canonicalized, or noindexed.
- [ ] `npm run lint` passes.
- [ ] `npx tsc --noEmit` passes.

## Current Known Gaps

These are the gaps identified during the May 2026 audit:

- Root layout metadata is generic and should become a proper sitewide fallback.
- Home page needs description, Open Graph, Twitter, and canonical metadata.
- `robots.ts` is missing.
- Several static pages have only title and description.
- Tool pages reference OG images that are not currently present in `public/`.
- Playbook pages have title and description only.
- Playbook pages are missing from `app/sitemap.ts`.
- Ask Rami, About, Contact, and tool pages are missing from `app/sitemap.ts`.
- Blog pillar pages have title and description only.
- Duplicate legal/policy routes need a canonical, redirect, or noindex decision.
